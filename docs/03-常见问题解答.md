# M3U8 下载助手油猴脚本 - 常见问题解答 (FAQ)

## 安装相关

### Q1: 在哪里下载 Tampermonkey？

| 浏览器 | 下载地址 |
|--------|----------|
| Chrome | [Chrome Web Store](https://chrome.google.com/webstore/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo) |
| Firefox | [Firefox Add-ons](https://addons.mozilla.org/firefox/addon/tampermonkey/) |
| Edge | [Edge Add-ons](https://microsoftedge.microsoft.com/addons/detail/tampermonkey/iikmkjmpaadaobahmlepeloendndfphd) |
| Safari | 需要使用 [Userscripts](https://apps.apple.com/app/userscripts/id1463298887) |

### Q2: 脚本安装后没有反应？

**检查步骤：**

1. 确认 Tampermonkey 图标上有数字徽章（表示启用的脚本数量）
2. 点击 Tampermonkey 图标，确认脚本已启用（开关为开启状态）
3. 刷新页面
4. 打开浏览器控制台（F12），查看是否有错误信息

### Q3: 提示"脚本需要以下权限：GM_xmlhttpRequest"？

这是正常的权限请求，点击 **「允许」** 即可。这个权限用于：
- 向后端服务发送下载请求
- 绕过同源策略限制

### Q4: 能否在手机上使用？

**Android：**
- 可以使用支持油猴的浏览器，如 Kiwi Browser、XBrowser 等

**iOS：**
- 可以使用 Userscripts + Safari
- 或使用支持油猴的第三方浏览器

## 使用相关

### Q5: 为什么没有出现浮动按钮？

**可能原因及解决方案：**

1. **页面中没有 M3U8 链接**
   - 不是所有视频都使用 M3U8 格式
   - 有些网站使用 MP4 直链或其他格式

2. **视频是动态加载的**
   - 等待视频开始播放
   - 脚本每 3 秒会自动扫描一次

3. **脚本被网站的安全策略阻止**
   - 查看控制台是否有 CSP (Content Security Policy) 相关错误
   - 尝试在其他网站测试

4. **iframe 限制**
   - 某些网站禁止脚本在 iframe 中运行
   - 尝试直接访问 iframe 的 URL

### Q6: 检测到的链接打不开或下载失败？

**可能原因：**

1. **防盗链保护**
   - 视频服务器检查 Referer 头
   - 后端服务已添加 Referer 伪造，但某些网站可能有额外验证

2. **链接已过期**
   - M3U8 链接通常有时间限制
   - 刷新页面获取新链接

3. **需要 Cookie 认证**
   - 某些视频需要登录状态
   - 后端服务不支持 Cookie 传递（未来可添加）

4. **IP 限制**
   - 服务器限制了 IP 访问
   - 需要使用代理或 VPN

### Q7: 下载速度很慢？

**可能原因及解决方案：**

1. **服务器限速**
   - 某些视频服务器限制了下载速度
   - 无解，只能等待

2. **并发数过低**
   - 修改后端服务的并发配置
   - `./m3u8_downloader serve --concurrent 16`

3. **网络问题**
   - 检查本地网络
   - 尝试使用下载工具测试服务器速度

### Q8: 如何下载 m3u8 以外的格式？

油猴脚本专门针对 M3U8 格式设计。对于其他格式：

- **MP4 直链**：可以直接右键另存为，或使用其他下载工具
- **FLV**：需要专门的 FLV 下载工具
- **流媒体加密**：需要更复杂的解决方案

### Q9: 进度弹窗卡住了怎么办？

**解决方法：**

1. 点击弹窗右上角的 ✖ 按钮关闭
2. 刷新页面重新操作
3. 如果弹窗无法关闭，执行以下代码：
   ```javascript
   document.querySelectorAll('[id^="download-progress"]').forEach(el => el.remove());
   ```

### Q10: 如何同时下载多个视频？

可以依次点击多个链接的下载按钮，每个下载任务会独立运行。后端服务支持并发下载多个任务。

## 配置相关

### Q11: 如何修改服务器地址？

编辑脚本文件，找到第一行配置：

```javascript
const BACKEND_URL = 'http://localhost:8080';
```

修改为你的服务器地址：

```javascript
const BACKEND_URL = 'http://192.168.1.100:8080';
// 或
const BACKEND_URL = 'https://your-server.com';
```

### Q12: 如何只在特定网站启用？

修改脚本头部的 `@match` 规则：

```javascript
// 只在 Bilibili 启用
// @match        *://*.bilibili.com/*
// @match        *://b23.tv/*

// 只在 YouTube 启用
// @match        *://*.youtube.com/*
```

### Q13: 如何禁用脚本？

**临时禁用：**
- 点击 Tampermonkey 图标
- 找到脚本，点击开关关闭

**永久禁用：**
- 在 Tampermonkey 管理面板中禁用或删除脚本

## 错误相关

### Q14: "GM_xmlhttpRequest is not defined"

**原因：** 脚本头部缺少 `@grant GM_xmlhttpRequest` 声明

**解决：** 检查脚本头部是否包含：
```javascript
// @grant        GM_xmlhttpRequest
```

### Q15: "Failed to fetch" 或 "Network Error"

**原因：** 无法连接到后端服务

**检查步骤：**
1. 确认后端服务正在运行
2. 确认 `BACKEND_URL` 配置正确
3. 尝试在浏览器中访问 `http://localhost:8080/api/tasks`
4. 检查防火墙设置

### Q16: "CORS policy" 错误

**原因：** 跨域请求被阻止

**解决：** 
- 使用 `GM_xmlhttpRequest` 而非 `fetch`（脚本已正确使用）
- 确认后端服务已启用 CORS

### Q17: WebSocket 连接失败

**原因：** WebSocket URL 配置错误或服务不支持

**检查：**
1. 确认服务支持 WebSocket
2. 检查 WebSocket URL 格式是否正确
3. 如果使用 HTTPS，需要使用 WSS 协议

## 高级问题

### Q18: 能否在无头浏览器中使用？

可以，但需要：
1. 安装 Tampermonkey 的无头版本
2. 或直接调用后端 API

示例（使用 curl）：
```bash
curl -X POST http://localhost:8080/api/download \
  -H "Content-Type: application/json" \
  -d '{"name": "test", "url": "https://example.com/video.m3u8"}'
```

### Q19: 能否批量下载整个播放列表？

**当前版本：** 需要手动点击每个视频

**未来版本：** 计划添加"批量下载当前页面所有链接"功能

**临时方案：**
1. 复制所有链接
2. 使用后端 API 批量创建任务

### Q20: 如何导出下载历史？

访问后端 API：
```bash
# 获取所有任务
curl http://localhost:8080/api/tasks

# 保存到文件
curl http://localhost:8080/api/tasks > tasks.json
```

## 其他问题

### Q21: 脚本安全吗？会泄露隐私吗？

**安全措施：**
- 脚本只检测 M3U8 链接，不会收集其他数据
- 只与本地后端服务通信
- 代码完全开源，可审查

**潜在风险：**
- 注入到页面的代码可能被恶意网站利用（概率极低）

### Q22: 与其他下载扩展冲突吗？

通常不会冲突。但如果有问题：
- 尝试禁用其他下载扩展
- 检查是否有多个脚本劫持了相同的 API

### Q23: 支持哪些 M3U8 类型？

| 类型 | 支持情况 |
|------|----------|
| 普通 M3U8 | ✅ 支持 |
| 主播放列表 (Master) | ✅ 支持（自动选择最佳流） |
| 加密 M3U8 (AES-128) | ✅ 支持 |
| 分段 M3U8 | ✅ 支持 |
| 直播 M3U8 | ⚠️ 部分支持（只能下载当前可用的片段） |

### Q24: 如何报告 Bug？

请提供以下信息：
1. 浏览器版本
2. Tampermonkey 版本
3. 脚本版本
4. 问题页面 URL
5. 控制台错误信息
6. 复现步骤
